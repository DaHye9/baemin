> 주어진 API 명세서의 기본 에러 케이스를 넘어, 실제 서비스 운영 단계에서 발생할 수 있는 다양한 오류 상황과 예외
케이스를 고민해보고, 이를 해결하는 과정을 자유롭게 서술해 주세요.

## 서버 (DB) 문제
* 서버나 네트워크 이슈가 생겼을 때 API 요청이 제대로 서버에 전달되지 않을 수 있습니다.
    - 서버의 엔드포인트를 막아 서버가 응답 가능하지 않은 경우를 시뮬레이션합니다.
    - 타임아웃 시간 이후에 서버가 응답하지 않는다는 오류 메세지를 띄우는지 확인합니다.
* scale test를 합니다. 동시에 트래픽을 많이 발생시켜 서버 부하가 걸렸을 때 모든 주문이 잘 들어가는지, 누락되는 주문은 없는지, 서버가 응답이 없을 때 잘 처리하는지 확인합니다.

## 동시성 문제
* 주문이 처리되는 과정에서 메뉴가 품절이 되는 경우, 서버에 요청이 들어온 순서대로 제대로 처리가 되어야 합니다. 
    - 한 메뉴의 재고를 1개로 설정하고, 동시에 두 개 이상의 동일한 메뉴에 대한 요청을 보냈을 때 하나의 요청만 성공하는지 확인합니다.
* 주문이 처리되는 과정에서 메뉴의 가격이 바뀌는 경우, 주문 요청이 성공하지 않아야 합니다.

## 요청 문제
* 동일한 주문이 동시에 중복으로 요청이 될 수 있습니다.
    - 동일한 요청을 여러 개 보냈을 때 하나의 요청만 처리되었는지 확인합니다.
* Code injection 공격을 받을 수 있는 문자가 포함되어 있을 수 있습니다.
    - `DROP TABLE`과 같은 공격 문자열을 모든 필드에 넣어 전송합니다. 잘못된 요청과 같은 응답이 오는지 확인하고, 데이터베이스에 변경된 사항이 없는지 확인합니다.
